LabIss2021 | Build and deploy the WEnv

Building

  1. Install Node.js
  2. Open a terminal on it.unibo.virtualRobot2020\node\WEnv\server and execute 
    npm install
  3. In the directory it.unibo.virtualRobot2020\node\WEnv\WebGLScene execute 
    npm install
  4. In the directory it.unibo.virtualRobot2020\node\WEnv\server\src execute 
    node WebpageServer.js
  5. Open a browser on 
    localhost:8090
Look at Dockerfile.

The components

Login in DockerHub (get the password from a private file) 
type docker_password.txt | docker login --username natbodocker --password-stdin

The Virtual robot project

gradle build                            Create the distribution
docker build -t virtualrobotdisi:1.0 .  Create the image using the Dockerfile

Tag the image
docker tag virtualrobotdisi:1.0 natbodocker/virtualrobotdisi:1.0

docker run -ti -p 8090:8090 -p 8091:8091 --rm  virtualrobotdisi:1.0 /bin/sh	local test


Register the image on DockerHub
docker push natbodocker/virtualrobotdisi:1.0

The docker-compose.yaml

The docker-compose.yaml defines two services:
  • wenv:
  • vrobotclient;
Run all

docker-compose up -d	                    vrobotclient is not able to connect (we should retry ...) 
docker-compose run --rm vrobotclient		starts also wenv if there is dependency


Run single services

docker-compose run -d --service-ports wenv   starts wenv 
docker-compose run --rm vrobotclient		 attempts to re-create wenv if there is dependency
docker-compose run --rm vrobotclient		 does not connect if there is NO dependency

docker run --rm docker.io/natbodocker/usingvirtualrobot:1.0		does not connect
